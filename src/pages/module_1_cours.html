Chapitre 1 – Cours : Les rôles du MVC

  • Model (Modèle) : stocke et gère les données de l’application.

  • View (Vue) : affiche les données à l’utilisateur.
  • Controller (Contrôleur) : relie le modèle et la vue et gère les actions de l’utilisateur.
    Exemple visuel : un blog → Model = articles stockés, View = page qui affiche les articles, Controller = bouton “Créer un article” qui déclenche l’action.
  • Aide mémoire : Modèle = cerveau, Vue = écran, Contrôleur = chef d’orchestre.
